# Contributing

## Code Style

1. Follow VNS prefix for all classes
2. Use `m` prefix for private properties
3. Use descriptive method names
4. Comment complex algorithms
5. Include examples in method documentation

## Testing Requirements

1. Test all public methods
2. Test error conditions
3. Test boundary cases
4. Test platform-specific code on both Desktop and Web

## Documentation Requirements

1. Update this file for API changes
2. Include usage examples
3. Document parameters and return values
4. Explain design decisions
